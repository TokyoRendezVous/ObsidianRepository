
存储类别说明符本质控制4件事：

|属性|说明|
|---|---|
|存储位置|栈 / 静态区 / 线程局部|
|生命周期|什么时候创建 & 销毁|
|作用域|在哪里可见|
|链接属性（linkage）|是否跨文件|

**C11、C17/C18常见的使用**

auto、register：几乎不用

## `static` —— 最重要的关键字之一

### （1）函数内部 `static`

```c
void func() {  
    static int counter = 0;  
    counter++;  
}
```
### 特点：

|属性|值|
|---|---|
|存储|静态区|
|生命周期|程序整个运行期间|
|作用域|函数内部|

👉 行为：

- 只初始化一次
- 多次调用共享状态

✔ 常见用途：

- 状态机
- 缓存
- 计数器

---

### （2）文件作用域 `static`

static int g_var = 10;

👉 关键作用：

> 🔒 **限制符号只在当前 .c 文件内可见**

---

### 对比：

int g_var;        // 全局可见  
static int g_var; // 仅本文件

---

### 🔥 工程意义（非常重要）

在大型项目 / SoC开发中：

👉 `static` = **模块封装**

✔ 避免：

- 符号污染
- 命名冲突
- 非预期访问

## `extern` —— 跨文件访问

```c
extern int g_var;
```

👉 含义：

> “这个变量定义在别的文件里，我这里只是声明”

---

### 示例

**file1.c**
```c
int g_var = 10;
```

**file2.c**
```c
extern int g_var;
```
---

### ⚠️ 常见错误

`extern int g = 10;` // ❌ 这是定义，不是声明
